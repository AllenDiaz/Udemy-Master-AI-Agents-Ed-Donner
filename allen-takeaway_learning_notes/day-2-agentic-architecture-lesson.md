# 📘 Day 2 Lesson: Agentic Architecture

## Overview
Day 2 focuses on **theory** rather than practice. The topic is **agents and agent architecture**, exploring definitions, hallmarks, and distinctions between workflows and agents.

---

## 🔑 Key Concepts

### **Definition of an Agent**
- According to Hugging Face:
  - **AI agents** are programs where **LLM outputs control the workflow**.
  - One LLM output decides tasks and their sequence.

### **Hallmarks of Agentic AI**
- **Multiple LLM calls** → chaining outputs.
- **Tool use** → LLMs interacting with external systems (e.g., turning on lights).
- **Coordination environments** → LLMs exchanging information.
- **Planners** → processes that coordinate activities.
- **Autonomy** → LLM decides order or actions (“choose its own adventure”).

---

## 📖 Important Definitions

- **Agentic AI**: Any system where LLMs have autonomy or orchestrate tasks.
- **Autonomy**: Granting LLMs control over task execution order.
- **Agentic System** (Anthropic):
  - **Workflows**: Predefined paths where models/tools are orchestrated.
  - **Agents**: Models dynamically direct their own processes/tools.

---

## ⚙️ Practical Examples

- **Tool Use**: LLM turning on lights (N810 demo).
- **Autonomy**: LLM choosing a business sector to analyze.
- **Workflow**: Multi-step orchestration with fixed paths.
- **Agent**: LLM dynamically deciding next steps in real time.

---

## 📝 Step-by-Step Process (Learning Flow)

1. **Understand the definition** → LLM outputs drive workflows.
2. **Identify hallmarks** → multiple calls, tool use, coordination, planners, autonomy.
3. **Distinguish categories**:
   - Workflows = predefined orchestration.
   - Agents = dynamic, autonomous control.
4. **Apply design patterns** → use autonomy and orchestration in real projects.
